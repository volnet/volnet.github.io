## 推荐序 德鲁克给我的十条教谕（吉姆·柯林斯）

吉姆·柯林斯，用自己的话，总结了十条教谕，清晰全面地概况的全书精要，每一小段都十分精辟，后续的段落，德鲁克将用它的智慧娓娓道来，期待开启后续的每一页。

## 前言

> 有效性是管理者获得报酬的原因，无论他们是需要对自己和他人绩效负责的经理（manager），还是只需要对自身绩效负责的专业的个体贡献者（individual professional contributor）。他们付出的聪明才智再多，花费的时间再多，如果有效性为零，“绩效”就为零。

1. 有效性（effectiveness），至关重要，它是管理者所必须的！
2. 卓有成效的管理者，不是天生的，只能通过后天习得，而且不难，遵守**为数不多的几种惯常做法**就可以获得这样的有效性。
3. 把这些惯常的做法变成习惯，就能够变得有效。

## 引言 有效管理者是由什么造就的

在作者熟知的一些优秀人士身上总结了8个惯常习惯（practice）：

- 他们会问“什么事是必须做的？”
- 他们会问“什么事对企业来说是正确的？”
- 他们制订行动计划
- 他们承担起决策的责任
- 他们承担起沟通的责任
- 他们专注于机会而不是问题
- 他们召开高效的会议
- 他们用“我们”而不是“我”来思考和说话

## 获得必需的知识

有效的管理者做事的原则：

1. 做必须做的优先级最高的事。
2. 做对企业（而不是股东、股价、员工或者管理者等）来说正确的事。

### 制订行动计划

> 行动计划是对意图的陈述，而不是死板的要求。不能让它成为束缚，而是应当经常修改，因为每一次行动，无论成败，都会带来新的机会。

行动计划是希望我们能够把事儿想清楚，但是计划里总是不能预估所有的变化，因此当变化出现的时候，我们应该积极进行调整。

### 行动

这一节用四个关键议题来描述：决策、沟通、机会（而非问题）和会议。

1. 决策：关键事项的负责人、截止时间、影响、相关人员。让有能力做决策的人去做这个决策，并定期进行评估决策的质量。决策不一定有效，但不决策注定无效。
2. 沟通：需要把行动计划和决策传达出去，同时要想办法把信息同步回来。得为这个结果去不懈努力。
3. 机会：只有新的机会会创造成果，解决问题最多只能降低损失。
4. 会议：两个及以上的人沟通其实就是开会，因此会议并非严谨的会议室会议。因此这里说的是有效利用时间。必须目的明确、事前准备、高效推进、跟进落实，总之，你不能漫无目的地浪费自己和他人的时间。

### 用“我们”来思考和说话

我们，是指把自己和所在组织视为一个整体，以组织的利益为优先，因为管理者之所以获得某种职权，是因为组织的利益需要他去这么做。

有效可以学会，也必须学会。

## 第1章 有效是可以学会的

> 才智、想象力和知识都是至关重要的资源，但只有有效性才能把它们转化为成果。至于这些资源本身，只是为能够取得什么样的成果设下了界限。

这句话一下把**有效性**和其他价值区分开来。

### 为何需要有效管理者

知识工作者区别于体力工作者，前者以思考为工作，而别人看不见摸不着，后者则有大量的评价工具产出数量、质量等来衡量，评价后者的方法对前者不适用。

> 因此，知识工作者必须做一件体力劳动者不需要做的事——提供有效性。他可不能指望自己的产出直接产生效用，就像一双做工精良的鞋子那样。

**提供有效性**，从原文的这个关键词看，知识工作者应该让自己“被看见”，这也是衡量他价值的重要方法。

### 谁是管理者

需要作出决策的知识工作者，都是管理者。它不因职位高低和大小而定。

### 管理者面对的现实

管理者想要有效的关键就是它们必须主动拥抱有效性，克服这些管理者必然面临的困境：

1. 时间被人占用：即便你很努力，可能你不得不接听上司或客户的电话，被占用很多与有效性无关的时间。
2. 忙于日常事务：应该做真正重要的事情，而不是按照事情发生的顺序去做事情。
3. 管理者更应该关注平行部门：知识工作者的有效性是由知识成果被他人使用来界定的，而使用这些成果的人是在外部，比如你的平行部门，而不是你的下属。
4. 组织内部无成果可言。成果全部存在于组织外部：因为只有客户/消费者可以为企业创造价值，它们在企业外部，且与企业无关。而管理者所接收的外部信息通常是有限的，管理者必须要主动想办法去关注外部，获取更多外部的信息。

### 有效带来的希望

虽然雇佣更优秀的人是个不错的建议，但是人类的大量历史证明，普通人还是占大多数，因此提高他们之间的协作效率是至关重要的。

现在的专业分工，每一个细分领域都可以深不见底，将不同的细分领域整合在一起则需要相互之间更加有效，我们不能寄希望于找到全才，因为它的供给太少了，但是我们可以寄希望于有效性，提高这些资源（专业的知识工作者，人，专才，而不是全才）的有效性将有助于提高整个组织的有效性。

### 有效真的可以学会吗

有效，作为一项本领，不依赖于天赋，而是习惯。它需要经过简单的一些实践，反复练习，并真正形成习惯。

而这些习惯，在后续的章节中逐一解释。

## 第2章 认识你的时间

资源决定完成工作的上限，时间是最稀缺的资源，且供给弹性为零，无法储存。有效管理时间，不是关注于工作计划，那只是美好的愿望，关注时间花在哪才能解决如何高效管理时间的问题。

### 管理者面对的时间需求

拥有大块的整段的时间才有可能取得有效性，而管理者的时间通常容易被碎片化，这个和管理者总是被各种琐事骚扰有关。要想要提高时间的有效性，可行的方法就是“记录时间被用在了哪里”。

### 时间诊断

把自己实际使用的时间，如实地实时的记录下来，而不是事后回忆。然后你会发现大多数情况下，时间的分配和你预想以及期望的分配情况有所不同。

接下来是系统性地管理时间：

1. 砍掉那些根本不需要做的，即做与不做毫无差别的事情。
2. 把那些可以让别人做的事情标记出来，也许别人做得比你更好。
3. 减少那些自己作为管理者而浪费别人时间的行为，一个简单的方法就是坦诚地问对方：我做的哪些事，对你的有效性没有贡献，所以是浪费你时间的？

### 精简浪费时间的活动

除了上面3种浪费时间的行为，还要防止管理和组织不善造成的时间浪费：

1. 缺少体系或者远见而浪费时间的活动：应该把经常出现的问题标准化，变成标准动作，而不是每次都需要一惊一乍。
2. 人员过剩导致的时间浪费：人多就容易产生内耗，那些为完成工作偶尔才需要的职能应该放在团队外面。
3. 组织结构缺陷导致的时间浪费：典型特征就是会议太多，每个人不能独立行事。会议在本质上是对组织结构有缺陷所做的补救。开会就不能工作，二者不可兼得。
4. 信息功能失灵导致的时间浪费：信息没有有效传递，不同人对相同的信息需要额外的时间去处理后才能得以利用，这都会造成时间的浪费。

### 集中“可自由支配的时间”

把时间集中起来利用，将可以大大提高有效性。

> 我至今没有碰到过什么危机，是不能等上90分钟再去处理的。

时间管理的最后一环就是把经过记录和分析之后发现的、通常属于管理者可以自由支配的时间集中起来。

时间是最稀缺的资源，不管好它，也就管不好任何其他事情。此外，时间分析是对工作加以分析，并且明确其真正重心的唯一既简单易行但又系统的方法。

## 第3章 我能做出什么贡献

> 一个眼里只有工作本身、看重向下权力的人，无论职位有多高，也只是一个“下属”。相反，一个重视贡献、对成果负责的人，无论职位有多低，都是名副其实的“高管”。他让自己对整个组织的绩效承担责任。

有效的管理者，更关注自己应该为组织承担什么责任，而不是工作本身。

### 管理者自己的追求

> 思考“我能做出什么贡献？”这个问题，就是探寻这份工作当中尚未得到发掘的潜力。就许多职位而言，如果把能做的贡献充分发掘出来，过去取得的那些绩效纵然辉煌，相比之下也会黯然失色。

有效管理者通常会问自己“我能做出什么贡献”，将它作为自己的工作目标。他将有可能变得卓有成效。

### 如何让专才有效

> 这意味着专才必须弄清楚谁会使用自己的产出，以及使用者必须知晓和理解哪些东西才能让自己生产的这些片段发挥成效。

让专才有效的办法就是他把自己的贡献和别人的工作结合在一起。他有义务让别人理解他的工作成果。

### 正确的人际关系

良好的人际关系：他们在处理自己的工作以及人际关系时都**专注于贡献**。这导致他们建立的关系富有成效。

相处融洽和言语和悦不仅没有意义，而且不过是恶劣态度的虚假门脸罢了。

**专注于贡献**本身可以满足有效人际关系必须具备的四个基本要求：

- 相互沟通：沟通不是上级要求下级，这样通常无效，而是让下级肩负责任。
- 团队合作：一般的专业人士在组织里从属于不同的专业部门，但团队里则需要他们和很多不同专业的人相互协作。
- 自我发展：对贡献的专注，才会带来自我发展。
- 培养他人：专注于贡献的管理者会让团队里的其他人也变得一样专注于贡献。

### 有效的会议

有效的会议，要有明确的目的，专注于解决具体的问题，而不应该是漫无目的的。

> 专注于贡献可以让管理者解决自己面临的一个基本难题：各种事务杂乱如麻，光看事情本身无法判断哪些重要，哪些纯粹是“噪声”。以贡献为标准就能做出判别。专注于贡献为管理者提供了组织原则，提供了判断事项相关性的标准。
>
> 专注于贡献还让管理者把工作情境中的一个先天劣势转变为优势之源：管理者需要依靠他人，且身处组织内部。专注于贡献可以让他把众多个体塑造成团队。

## 第4章 让长处富有成效

众人的长处才是真正的机会所在。

### 用人所长

用人就是要用人所长，容人所短。组织把不同的人的长处整合起来才有可能达到卓越，否则将趋于平庸。

即便是用人所长也不应因人设岗，而要因任务设岗，为此要遵循以下四条原则：

1. 避免设计“无人胜任”的岗位，有一些岗位被设计成只有天才才能胜任，导致不论这个岗位换了几任都难以达到要求，这时候问题通常是因为岗位设计本身出了问题。
2. 每个职位都要设计成有足够挑战的。因为所设计的岗位所面临的问题通常都会发生变化，所以此时此刻的完美匹配随着面对的挑战发生变化就会变得不完美。
3. 在考评员工的时候，应该充分考评他有什么长处以及他可以用于何处，而不是关注他有什么不足，并不切实际地期望他补足不足。
4. 要用人所长且能容人缩短，瑕不掩瑜。

用人所长不要受限于这个人离不开某个岗位，比如一个管理者认为自己有一个不可替代的下属时，恰恰说明这里存在管理问题，如果这个下属不可替代，那么也恰恰证明他在补足他本人的不足。

始终用人所长并以此为纲，则可以让组织不断培养出优秀的人。

### 管理上级

上级也是人，有他的长处和短处。所以让上级有效：

1. 发挥上级的长处；
2. 用合适的方式提出，有的上级喜欢听、有的上级喜欢看报告，有的上级喜欢一开始就插手有些则相反；

### 让自己有效

了解自己是让自己有效的关键之一，成年人很多已经形成了自己的性情和习惯，就像是每个人都清楚自己更擅长使用左手还是右手，他们接收信息的方式是听还是看、喜欢脱稿演讲还是需要提前准备、喜欢和人沟通还是独自行事，结合自身的特点加以利用。

看到困难并把它挂在嘴边总是容易的，但是这只是找到问题但是并不解决问题，要始终关注机会而不是问题。用人所长就是一种关注机会的做法，因为机会是通过长处去把握的而不是短处。

## 第5章 要事优先

每次只做一件事，而且要事优先。很少有人能同时能做三件事，那些看似同时做多件事且保持有效的人，每做一件事的时候，都给它分配达成某个进度所需要的最小整块时间。

### 抛弃昨天

> 管理者聚焦的首要法则，是抛弃那些已经不再有成效的旧事。

有效的管理者总是应该重新审视计划列表的任务，并确认它值得去做，果断删掉那些已经过时的无关紧要的事项。

在引入一项新业务的时候，应该剔除一项旧任务，否则整个组织将很快陷入资源不足的困境。

在引入新人的时候，要尽可能避免将他放在新的有战略意义的任务上，而是让过去已经被证明卓有成效的人去做这项新任务，而让这个新人去接替旧的任务。

### 优先任务与延迟任务

> 但是，真正难的还不是确定优先任务。这做起来容易，谁都能办到。之所以很少有管理者能够做到聚焦，原因在于很难设置“延迟任务”，也就是决定哪些事情不去做，并严守这个决定。
>
> 有效管理者会全力以赴去完成当前聚焦的那一项任务。在完成该项任务之后，他再对形势做出评估，确定接下来哪一项任务最重要。

每次只聚焦一件优先任务事关重要，并要有勇气放弃一些延迟任务。

## 第6章 决策的要素

> 管理者由于担任职务和拥有知识，于是被人期待做出对整个组织及其绩效和成果有重大影响的决策。
>
> ……
>
> 有效管理者做的决策不会很多，他们聚焦于那些重要的决策。他们会努力想清楚那些具有战略性和共性的东西，而不是想着怎么“解决问题”；努力站在概念性理解（conceptual understanding）的最高层面做好为数不多的重要决策；努力找出在某个情境下的常量。因此，他们不会有太大的压力要快速做出决策。实际上，他们认为精于操控众多变量是思考不够深入的表现。他们希望弄清决策到底是怎么回事，还有决策需要充分考虑哪些根本的现实情况。他们要的是收到实效，而不是提高技巧；要的是明智，而不是聪明。

1. 只做关键决策；
2. 关注战略性和共性的东西，而不是解决问题；
3. 关注常量，也就是那些不变的东西；
4. 做出决策只是其中一步，更重要的是落实；

### 两个决策案例

决策案例1：西奥多·韦尔（Theodore Vail）为了实现贝尔公司“想保持私营身份和拥有自主管理权”，为此他做出了四个关键决策：

1. 确定贝尔公司的业务就是服务：必须把预见和满足公众的服务需求确立为公司的业务。——这样才能超出国有化公司的水平。
2. 主动拥抱公共监管：一家全国性的垄断电信企业不可能成为传统意义上的自由企业，即自主管理和不受政府干预的私营企业。
3. 成立贝尔实验室：用未来的产品和技术与现在的产品和技术竞争，不论现在的产品有多么赚钱。
4. 发明了大众资本市场：从大众手里募资。

决策案例2：通用汽车的斯隆设计的分权管理制度，在确保大企业有掌控权的情况下，充分发挥人员的积极性。

这些现在看来司空见惯的决策，在当年很多人的常识里都是天方夜谭，之所以他们能够做出这样的决策，恰恰是因为它们都是在概念性理解的最高层面去解决问题。他们决策之前都是先弄清问题的本质，再努力提出决策的指导原则。换句话说，这些决策都具有战略性，而不是针对当时表面上的需要所做的适应性调整。它们都是创新性的，都极富争议。事实上，他们的这五个决策完全有悖于当时“人人都知道”的道理。

### 决策过程的要素

1. 认清这是一个共性问题且只有通过建立规则和原则的决策才能解决。
    - 很多问题实际上并不需要真正意义上的决策，它只是在现有制度下做出新的调整，就像一些工程问题一样。
    - 对你而言是偶尔一次的，但是有别人的经验可以借鉴，这种实际上也不需要真正的决策。
    - 真正的例外，仅发生一次的。
    - 共性的问题，需要建立规则和原则的决策，来系统性地解决。
2. 定义解决方案需要满足的参数（specifications），即“边界条件”（boundary conditions）。
3. 先想清楚什么是“正确的”，也就是完全满足各项参数的解决方案，再去思考让人接受决策需要做出哪些妥协、调整和让步。
    > 我对你只有一个要求，那就是你认为什么是正确的，就如实写下来。不要担心我们的反应。不要担心我们是不是喜欢。最重要的是，不要去想为了让你的建议得到采纳，需要做出哪些妥协。只要有人提出必须做出妥协，我们公司没有一个管理者，是在没有你的帮助的情况下，就不知道怎样去做这个妥协的。但是，除非你先告诉他们什么是‘正确的’，他们才能做出‘正确的’妥协。”
4. 把落实决策所需采取的行动融进决策。只有找到有能力执行决策的人，决策才算完成。
5. 建立“反馈机制”，用事情的实际发展情况检验决策的有效期和有效性。只有自己走出去，亲临现场，而不是光听报告，才有可能获得真实的反馈。

## 第7章 有效的决策

决策，就是在多个方案中做选择，如果只有一个方案，就算不上做决策。

为了做出正确的决策，有效管理者会要求至少多个不同的观点和意见，大部分情况下不是哪一条固然正确，而是当不同意见出现时，有效管理者会追问：

> 这个人之所以得出这样一个明显错误的结论，是因为看到了不同的现实，关心的是不同的问题。所以，有效决策者总是会问：“这个人要看到哪些东西，他的立场才是站得住脚的、理性的、明智的？”也就是说，他首先关心的是理解对方，然后才去想谁对谁错。

决策并非一定带来改变，决策的结果有可能是根本不需要决策，当下的情况或许是最好的情况，问题可能并不要强势干预就自己能够被解决。

决策就像动手术，没有医生会去动一个没有必要去做的手术，决策必然会对现有的系统产生影响，这是有代价的。

### 决策与计算机

计算机的出现让人们误认为计算机会替代人做决策，但事实上并不会，它只会呈现事实。

但是它的出现迫使人们做出决策，甚至一些原本不需要决策人，在计算机的帮助下也有能力且必须在工作中做决策。

## 第8章 结语：有效是必须学会的

本章按照全书脉络整理和回顾了全书的主要观点。

在发达国家，随着工资的增长，人们会有自我实现的需求，而组织的目标并不会自动转化为个人的目标，人们通过组织实现个人对自我实现的需求，而不仅仅是为了满足生计而奔波。而“有效性”就是同时融合组织需求和个人需求的重要手段。
